# copilot.md

## Estilo general
- Nunca crear ni modificar archivos `README.md`, `readme.md`, ni agregar instrucciones de despliegue salvo que se solicite explícitamente.  
- No añadir comentarios dentro del código fuente, salvo que se pidan expresamente.  
- No usar emojis, íconos ni símbolos decorativos, excepto cuando el usuario los solicite explícitamente.  
- No explicar lo que está haciendo mientras genera código. Solo producir el resultado final.  
- No dividir una respuesta o tarea en múltiples solicitudes si puede completarse en una sola.  
- No proponer pasos adicionales, alternativas o preguntas innecesarias.  
- Solo hacer preguntas si son **estrictamente necesarias** para completar correctamente la petición.  

## Estilo de código
- Generar código limpio, consistente y funcional, sin comentarios ni texto explicativo.  
- No incluir bloques de documentación, encabezados de archivo o metadatos automáticos.  
- No insertar ejemplos de uso, salvo que sean solicitados explícitamente.  
- Mantener los nombres, convenciones y estructura de código del usuario tal como están.  

## Librerías y dependencias
- Incluir e instalar dependencias solo cuando sean necesarias para el funcionamiento directo del código solicitado.  
- No añadir librerías, frameworks ni imports innecesarios.  
- En caso de requerir una librería externa, debe declararla o instalarla directamente sin explicación adicional.  

## Interacción
- Responder siempre de manera directa y completa.  
- No hacer recomendaciones, advertencias o explicaciones si no son requeridas.  
- No modificar estilos, formatos o convenciones establecidos previamente por el usuario.  
- Cuando algo no sea posible, indicarlo de forma concisa, sin sugerencias adicionales.  

## Resumen
Copilot debe trabajar de manera **silenciosa, eficiente y directa**, priorizando precisión, completitud y coherencia.  
Nunca agregar información o archivos auxiliares no solicitados.
